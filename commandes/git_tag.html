Git tag

Cette commande permet d'afficher toutes les étiquettes référencées dans refs/tags/. Elles s'affichent par ordre alphabétique.

Syntaxe

git tag

Affichage

v1.0
v2.0
v3.0
etc..

git tag -a

Cette commande permet de créer des étiquettes annotées. En utilisant seulement cette commande, Git ouvre l'éditeur pour spécifier le message d'étiquetage qui sera stocké avec l'étiquette (voir plus loin pour éviter cela). Utilisez ensuite git tag pour afficher l'étiquette créée.

Syntaxe

git tag -a v1.0
git tag

Affichage

v1.0

git tag -d

Cette commande permet de supprimer l'étiquette choisie

Syntaxe

git tag -d v1.0

Affichage

Deleted tag 'v1.0' (was a545472)

git tag -f

Cette commande permet de modifier le message d'étiquetage stocké avec une étiquette existante. Git va ouvrir votre éditeur pour le modifier.

Syntaxe

git tag -a v1.0 -f

Affichage

Updated tag 'v1.0' (was 5d04ass)

git tag -m

Cette commande permet de spécifier le message d'étiquetage qui sera stocké avec l'étiquette.Si vous ne spécifiez pas de message en ligne pour une étiquette annotée, Git lance votre éditeur pour pouvoir le saisir. Pour afficher ce message, entrez la commande git show gitname

Syntaxe

git tag -a v1.0 -m Etiquette
git show v1.0

git push --tags

Si vous avez de nombreuses étiquettes que vous souhaitez pousser en une fois, vous pouvez utiliser la commande git push --tags. Ceci transférera toutes les nouvelles étiquettes vers le serveur distant.

Syntaxe

git push --tags
