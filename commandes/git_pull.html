Git pull

Permet de mettre a jours votre copie locale pour etre sûr de travailler sur la derniere version et eviter les conflits.

Syntaxe

git pull --rebase

Utlisation

fait un git rebase a la place d'un git merge.
Exemple de l'affichage de la commande dnas le terminal

git pull --ff-only

refuse de merge et quitte si le HEAD n'est pas a jour et peut être résolu par un fast forward. .

Affichage

Exemple de l'affichage de la commande dnas le terminal

git pull --no-commit

réalise un merge mais prétend qu'il a raté et ne commit pas automatiquement.Cela donne une chance a l'utilisateur d'inspecter le merge avant le commit. .

Affichage

Exemple de l'affichage de la commande dnas le terminal

git pull --no-ff
Crée un commit de “merge” qui aura pour parents notre commit de modification, et le dernier commit connu sur la branche, même si il n’y a pas eu de modification sur cette dernière. .
