git rebase

la fonction git rebase est là pour changer le commit de base d’une branche, c’est-à-dire son point de départ.

Syntaxe :

git rebase master gh-pages
First, rewinding head to replay your work on top of it...
Applying: word 1
Applying: word 2
Applying: word 3

git rebase --abort

Cette fonction permet d'interrompre l'opération de rebase et de remettre le HEAD à la branche originale. Si la a été fournie lors du commencement du rebase, alors le HEAD sera remis à la . Sinon, le HEAD sera remis à où il était quand l'opération de rebase a été commencée.

Syntaxe :

git rebase --abort
C:\Users\Adam\Documents\GitHub\test [:REBASE +0 ~0 -0 !1 : +0 ~0 -0 !1]> git rebase --abort
C:\Users\Adam\Documents\GitHub\test [gh-pages]>

git rebase --onto

Cette commande permet de redéfinir un nouveau point d'origine pour tout les nouveaux commits. Ce nouveau point peut être initialisé à partir de n'importe quelle commit valide.>

Syntaxe :

git rebase --onto
First rewinding head to replay your work on top of it...
+Applying: F2a
+Applying: F2b

git rebase -i

Cette commande présente une liste de tous les commits qui sont sur le point d'être "rebase". Cela permet à l'utilisateur de modifier la liste avec qu'elle ne soit "rebase". Cette commande peut également être utilisée pour séparer certains commits.

Syntaxe :

pick 01d1124 Adding license +pick 6340aaa Moving license into its own file +pick ebfd367 Jekyll has become self-aware. +pick 30e0ccb Changed the tagline in the binary, too. + +# Rebase 60709da..30e0ccb onto 60709da +# +# Commands: +# p, pick = use commit +# e, edit = use commit, but stop for amending +# s, squash = use commit, but meld into previous commit +# +# If you remove a line here THAT COMMIT WILL BE LOST. +# However, if you remove everything, the rebase will be aborted.
Exemple d'affichage dans le terminal

git rebase --continue
