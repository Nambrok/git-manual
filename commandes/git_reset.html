git reset

Descriptif de la commande détaillé

git reset --soft
Cette commande ne touche pas le fichier d'index ( zone de transit où les nouveaux commits sont préparés) ou même l'espace de travail (les fichiers qui sont en train d'être modifiés) mais réinitialise le dernier commit, comme toutes les options de git reset.
Les fichiers modifiés restent des «Changes to be commited» (ou changement qu'il faut commit), comme git status dirait si bien.




Syntaxe

git reset --soft

Par exemple, pour défaire et refaire un commit

git commit ... git reset --soft HEAD^ edit git commit -a -c ORIG_HEAD

git reset --mixed

Cette commande permet d'enlever de la file d'attente une modification qui attend d'être committé. Les modifications restent intact mais ne sont plus marqué comme étant en attente d'être committé.

Syntaxe

git reset --mixed
$ git add Nouveau_dossier.html
$ git status
On branch master
Changes to be committed:
new file : Nouveau_dossier.html
 $ git reset --mixed
$ git status
On branch master
Untracked files :
(use git add < file >..." to include in what will be committed)
Nouveau_dossier.html
nothing added to commit but untracked files present (use "git add" to track)

git reset --hard

Supprime toutes les modifications par rapport au dernier commit. Cela revient au même que d'entrer la commande reset --hard HEAD.
La commande reset --hard HEAD^ supprime le dernier commit.
La commande git reset --hard v1.0 supprime tous les commits depuis la v1.0.

Syntaxe

git reset --hard
HEAD is now at 20cab59 Mege pull request #59 from yobero/gh-pages
